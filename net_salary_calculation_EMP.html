Forward Calculation — How to Get to Sal.Net
Read top to bottom. Each step feeds the next. All percentages apply to Base TPR (col M) — not to Sal.Base directly. The only exception is Mituel, which uses a sub-total after the statutory deductions.
1

Sal.Base — The Starting Point

Col J · Input from contract  |  Col K · J.Trav. (working days)  |  Col L · Div.Tax. (taxable supplement)

Sal.Base (col J) Fixed monthly amount from employment contract
J.Trav. (col K) Number of working days — 31 for all SIEGE staff
Div.Tax. (col L) Taxable supplement — 0 for all current SIEGE employees
2

Base TPR — The Tax Base (col M)

Col M = J + L  ·  Every deduction is calculated from this number

Base TPR = Sal.Base + Div.Tax. Excel: =J+L
Because Div.Tax. = 0 for all SIEGE employees, Base TPR = Sal.Base in every case. The distinction only matters if a taxable supplement is ever added in col L.
3

Statutory Deductions on Base TPR — Employee pays

Cols N, O, P · Calculated automatically from Base TPR · Always deducted

RSSB 6% employee pension  col O = Base TPR × 6%  ·  Excel: =M*6/100
RSSB 0.3% employee medical  col P = Base TPR × 0.3%  ·  Excel: =M*0.3/100
TPR — Tax on Professional Revenue  col N Progressive rate on Base TPR (see brackets below)
Base TPR range (RWF)RateTax in this bracketExcel formula segment
0 – 60,0000%0IF(M<=60000, 0, …)
60,001 – 100,00010%(M − 60,000) × 10%(M-60000)*10%
100,001 – 200,00020%4,000 + (M − 100,000) × 20%((M-100000)*20%)+4000
Above 200,00030%24,000 + (M − 200,000) × 30%((M-200000)*30%)+24000
Full Excel formula (col N): =IF(M<=60000,0,IF(AND(M>60000,M<=100000),(M-60000)*10%,IF(AND(M>100001,M<=200000),((M-100000)*20%)+4000,((M-200000)*30%)+24000)))
4

Mituel 0.5% — Community Health Insurance (col Q)

Calculated on the salary that remains after Steps 2 & 3 — not directly on Base TPR

Mituel base = Base TPR − TPR − RSSB 6% − RSSB 0.3% (M − N − O − P)
Mituel 0.5% = Mituel base × 0.5% Excel: =(M-N-O-P)*0.5/100
Mituel is applied after the statutory RSSB and income tax are removed. This avoids a circular calculation — the community health deduction is taken from what is left, not from the gross.
5

Other Variable Deductions — Input per employee

Cols R, S, T, U, V · Fixed amounts entered manually — varies per person

Cot.Twis.  col R Monthly savings contribution to Twis fund — fixed amount
Remb.Tw.  col S Monthly repayment of a Twis loan
Cot.Social  col T Social contribution — specific scheme
Avance  col U Salary advance recovery — amount already paid to employee earlier
Assur.  col V Insurance premium deduction
These are all input values — there is no formula. The HR manager enters the correct amount for each employee each month. If none apply, the cell is blank (treated as 0).
6

Sal.Net — Final Take-Home Salary (col W)

=ROUND((M−N−O−P−Q−R−S−T−U−V), 0) · Rounded to nearest whole RWF

Base TPR  col M+
− TPR (income tax)  col N−
− RSSB 6% employee pension  col O−
− RSSB 0.3% employee medical  col P−
− Mituel 0.5% community health  col Q−
− Cot.Twis. (savings)  col R−
− Remb.Tw. (loan repayment)  col S−
− Cot.Social  col T−
− Avance (advance recovery)  col U−
− Assur. (insurance)  col V−
= Sal.Net  col W = ROUND(M−N−O−P−Q−R−S−T−U−V, 0)= ?
★

Employer RSSB Contributions — NOT deducted from employee

Cols F, G, H · Company's own cost · Tracked for reporting only

RSSB 6% employer pension  col F = Base TPR × 6%  ·  Excel: =M*6/100
Hasard 2% occupational hazard  col G = Base TPR × 2%  ·  Excel: =M*2/100
TOTAL RSSB 8%  col H = col F + col G  ·  Excel: =F+G
These three columns appear in the sheet so the company can track its total payroll cost. They are never subtracted from Sal.Net. The employee does not see this deduction — the employer pays it separately to RSSB.
Note on col I (RSSB 0.3%): Between the employer block (F, G, H) and Sal.Base (J), the sheet has a col I also labelled "RSSB 0.3%". It has the same value as col P and uses the same formula =M*0.3/100. It is a reference / display column only — it does not appear in the Sal.Net formula and is not deducted anywhere.
